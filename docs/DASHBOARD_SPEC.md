# OpenFi Dashboard 功能规格文档

## 1. 整体设计

### 1.1 主题
- **深色模式**: 纯黑背景 (#000000)
- **浅色模式**: 纯白背景 (#FFFFFF)
- **多彩模块**: 每个模块使用不同的强调色

### 1.2 自动保存
- 用户修改任何设置后自动保存到后端
- 无需手动点击保存按钮
- 显示保存状态提示（保存中/已保存）

### 1.3 API 测试
- 所有 API 配置旁边都有测试按钮
- 点击测试按钮验证 API 连接和响应
- 显示测试结果（成功/失败/错误信息）

## 2. 情报模块

### 2.1 首页显示
- 显示最新 3 条情报
- 实时更新
- 比其他模块大 1/3（占 8 列 vs 4 列）

### 2.2 详情页面
#### 上方：最新情报事件
- 默认显示最多 3 条
- 点击"查看更多"可以看到更多情报
- 每条情报显示：标题、时间、来源、重要性

#### 下方：本周交易日历
- 按周历模式排版
- 显示经济数据发布时间
- 标注重要性级别

#### 右上角设置按钮
- **新闻 API 配置**
  - API Key 输入
  - 测试按钮
  - 启用/禁用开关
  
- **社交媒体舆情 API**
  - Twitter/X API
  - Reddit API
  - 其他社交媒体源
  - 测试按钮
  
- **关键词设置**
  - 添加/删除关键词
  - 关键词分类
  
- **权重设置**
  - 不同来源的权重
  - 不同关键词的权重

## 3. 行情模块

### 3.1 特别关注（3个）
- 用户最多选择 3 个特别关注的行情
- 这 3 个行情会在快讯和推送中附带现价
- 只有这 3 个可以进行量化回测

### 3.2 关注列表（10个）
- 普通关注列表，最多 10 个

### 3.3 研报机会留意（50个）
- 研报关注列表，最多 50 个

### 3.4 分类管理
- 商品
- 外汇
- 债券
- 股票
- 加密货币

### 3.5 搜索添加
- 支持搜索品种名称
- 支持搜索代码
- 添加到对应分类

### 3.6 K线图功能
#### 点击特别关注的行情
- 打开 K线图界面
- 齿轮按钮设置：
  - 接入第三方网站 HTML
  - 接入 MT4/MT5 行情
  
#### 量化记录叠加
- 如果有该行情的量化记录
- 将回测记录叠加在 K线上
- 显示买卖点、止损止盈等

## 4. 量化引擎模块

### 4.1 EA 库管理
- 上传 EA 文件（按钮）
- 重命名 EA
- 删除 EA
- 最多可以拖动 10 个 EA 到回测规则

### 4.2 因子库管理
- 上传因子文件（按钮）
- 重命名因子
- 删除因子
- 最多可以拖动 50 个因子到回测规则

### 4.3 回测设置
- 设置关注品种或股名关键词
- 从量化库拖动：
  - 10 个 EA（置顶）
  - 50 个因子规则
- 执行日报或研报回测

### 4.4 回测限制
- **只能对特别关注的 3 个行情进行回测**
- 回测结果可以叠加到 K线图上

## 5. AI Agents 模块

### 5.1 Agent 管理
- 最多创建 5 个 Agent
- 添加/删除 Agent
- 每个 Agent 可以独立配置

### 5.2 Agent 功能选择
每个 Agent 可以勾选以下功能：
- 新闻分析
- 行情监控
- 量化分析
- 事实核查
- 选股推荐
- 研报生成
- 或全部选择

### 5.3 Agent 通报设置
- **快讯通报**
  - 启用/禁用
  - 通报频率
  - 通报渠道（Telegram/Discord/Email等）
  
- **研报时间**
  - 设置研报生成时间
  - 每日/每周/自定义
  
- **日报规则**
  - 设置日报生成规则
  - 包含内容选择

### 5.4 回测前置选项
- 设置是否在推送研报/日报前先用量化引擎回测
- 回测结果梗概包含在报告中

## 6. 实盘账户模块

### 6.1 密码保护
- 点击后需要输入密码
- 超过 60 秒无响应需要重新输入密码
- 密码验证通过后才能查看详情

### 6.2 账户管理
- 最多添加 3 个实盘账户
- 设置账户类型：模拟/实盘
- 查看账户状态：
  - 是否运行 EA
  - 当前仓位
  - 资金余额

### 6.3 强制停止
- "强制停止所有"按钮
- 需要弹框二次确认
- 停止所有 EA 和自动交易

### 6.4 Agent 权限设置
- 默认所有 Agent 可以读取账户（不可交易）
- 单独开启某个 Agent 可以交易
- **每个账户只能有一个 Agent 下单交易**
- 设置交易模式：
  - 需要用户手动开启量化
  - Agent 全权接管

### 6.5 风控设置
- 最大止损设置
- 强平阈值设置
- 单笔最大交易量
- 日内最大亏损

### 6.6 API 连接（齿轮按钮）
- 输入 API Key
- 输入 API Secret
- 选择交易所/券商
- 测试连接按钮

## 7. 设置模块

### 7.1 基础设置
- **时区设置**
  - 选择用户所在时区
  - 影响所有时间显示
  
- **UI 语言**
  - 中文/英文切换
  - 改变 UI 语言同时改变 Bot 通知语言
  - 不更新用户设定的关键词、行情等

### 7.2 翻译设置
- **快讯翻译**
  - 原文/翻译/双语
  - 后台统一处理，不重复推送
  - 大众熟知的术语和行情简称不翻译

### 7.3 数据管理
- **用户数据下载**
  - 下载所有配置
  - 下载历史记录
  - 下载回测结果
  
- **重置系统**
  - 需要输入密码两次
  - 需要二次确认
  - 清空所有数据和配置

## 8. 推送规则

### 8.1 特别关注推送
- 快讯和经济数据预警发布时
- **只推送特别关注的 3 个行情的现价**
- 其他行情不附带现价

### 8.2 推送内容
- 快讯标题和摘要
- 相关行情现价（仅特别关注）
- 重要性级别
- 时间戳

### 8.3 推送渠道
- Telegram
- Discord
- Email
- 飞书/钉钉
- 微信（如果可用）

## 9. 技术实现要点

### 9.1 自动保存机制
```javascript
// 监听所有输入变化
input.addEventListener('change', async (e) => {
    showSavingStatus();
    await saveToBackend(e.target.value);
    showSavedStatus();
});
```

### 9.2 API 测试
```javascript
async function testAPI(apiType, apiKey) {
    showTestingStatus();
    const result = await fetch('/api/v1/test-connection', {
        method: 'POST',
        body: JSON.stringify({ type: apiType, key: apiKey })
    });
    showTestResult(result);
}
```

### 9.3 K线图集成
- 使用 TradingView Widget
- 或接入 MT4/MT5 WebTerminal
- 或自定义 HTML iframe

### 9.4 回测结果叠加
- 使用 Chart.js 或 ECharts
- 在 K线图上绘制买卖点标记
- 显示止损止盈线
