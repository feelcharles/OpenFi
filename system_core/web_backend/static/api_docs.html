📚 OpenFi API 文档

完整的 REST API 参考文档

💡 提示： 如果 Swagger UI 无法加载，可以使用此简化版文档。 大部分端点需要 JWT 认证，请先调用登录接口获取 Token。

🔐 认证 (Authentication)

POST /api/v1/auth/login

用户登录，获取 JWT Token
请求体： {"username": "admin", "password": "your-password"}
响应： {"access_token": "...", "token_type": "bearer"}

POST /api/v1/auth/logout 需要认证

用户登出

POST /api/v1/auth/register

注册新用户
请求体： {"username": "...", "email": "...", "password": "..."}

⚙️ 配置管理 (Configuration)

GET /api/v1/config/{file_name} 需要认证

获取配置文件内容
可用文件： llm_config.yaml, fetch_sources.yaml, push_config.yaml 等

PUT /api/v1/config/{file_name} 需要认证

更新配置文件
请求体： {"content": "YAML格式的配置内容"}

👥 用户管理 (Users)

GET /api/v1/users/me 需要认证

获取当前用户信息

GET /api/v1/users 需要认证

获取所有用户列表（需要管理员权限）

💰 交易管理 (Trades)

GET /api/v1/trades 需要认证

获取交易列表
查询参数： limit, offset, status

GET /api/v1/trades/{trade_id} 需要认证

获取单个交易详情

📊 仪表板 (Dashboard)

GET /api/v1/dashboard/metrics 需要认证

获取系统指标数据

GET /api/v1/dashboard/system-status 需要认证

获取系统状态

🔧 系统端点 (System)

GET /health

健康检查端点（无需认证）

GET /openapi.json

获取 OpenAPI 规范（无需认证）

📖 使用说明

  1. 首先调用 POST /api/v1/auth/login 获取 Token
  2. 在后续请求的 Header 中添加: Authorization: Bearer YOUR_TOKEN
  3. 使用上面的"测试此接口"按钮可以快速测试端点
  4. 完整的 API 规范请访问: /openapi.json
